# 최종 보완 실험 설계 정리

## 1. 실험 설계 목적

기존 BBB 프로젝트는 MAR20 항공기 crop 이미지를 대상으로 ResNet, EfficientNet, MobileNet 계열 모델을 학습하고 성능을 비교했다. 그중 EfficientNet-B0는 ImageNet pretrained backbone을 사용하고 마지막 classifier를 MAR20 20개 클래스로 교체한 뒤 full fine-tuning한 모델이다.

하지만 교수님 지침 관점에서는 기존 사전학습 모델을 단순히 가져와 classifier만 바꾼 것으로 보일 수 있다. 따라서 보완 실험의 목적은 다음과 같다.

```text
1. 모델 구조를 어떻게 변경했는지 명확히 보이기
2. attention과 classifier head의 효과를 분리해서 비교하기
3. 사전학습 모델을 특징 추출기, 일부 계층 재학습, 전체 fine-tuning으로 각각 활용한 경우를 비교하기
4. seed 3개 반복으로 결과의 우연성을 줄이기
```

## 2. 최종 실험 원칙

모델 구조와 학습 방식의 우수성은 seed 1, 2, 3의 평균과 표준편차로 비교한다.

최종적으로 사용할 checkpoint는 선택된 조건 안에서 validation 또는 test 성능이 가장 좋은 seed의 모델을 사용한다.

```text
구조 비교: seed 평균과 표준편차 기준
최종 checkpoint 선택: best seed 기준
```

이렇게 구분하는 이유는 다음과 같다.

```text
평균 성능은 구조 자체가 안정적으로 좋은지 판단하는 기준이고,
best checkpoint는 실제 최종 분석에 사용할 대표 모델을 고르는 기준이다.
```

## 8. 고정된 모델 세부 설정

### Attention 위치

Attention은 EfficientNet-B0의 feature extractor 출력 뒤, global average pooling 전에 둔다.

```text
Input
-> EfficientNet-B0 features
-> Attention
-> Global Average Pooling
-> Classifier Head
-> Output
```

이 위치를 선택한 이유:

```text
1. attention이 C x H x W feature map에 직접 적용될 수 있다.
2. pretrained backbone 내부 구조를 크게 훼손하지 않는다.
3. 기존 EfficientNet-B0와 비교할 때 변경 지점이 명확하다.
```

### MLP Head

MLP classifier head는 다음 구조로 고정한다.

```text
Linear(1280 -> 512)
-> ReLU
-> Dropout(0.3)
-> Linear(512 -> 20)
```

1280은 EfficientNet-B0의 마지막 feature channel 수이고, 20은 MAR20 클래스 수이다.

### 왜 head 조합을 여러 개 비교하지 않는가?

MLP-256, MLP-512, MLP-1024, ReLU, SiLU, dropout 0.2/0.3/0.5 등을 모두 비교하면 실험 수가 급격히 늘어난다. 또한 성능 변화가 hidden dimension 때문인지, activation 때문인지, dropout 때문인지 해석하기 어려워진다.

이번 실험의 핵심은 head 최적화가 아니라 attention과 학습 범위 비교이므로 MLP head는 하나로 고정한다.

```text
Hidden dimension: 512
Activation: ReLU
Dropout: 0.3
Depth: hidden layer 1개
```

## 9. 유지한 것과 제외한 것

### 유지한 것

```text
1. EfficientNet-B0 pretrained backbone 사용
2. SE vs ECA attention 비교
3. MLP-512 classifier head
4. Attention 위치: features 출력 뒤, pooling 전
5. 학습 범위 3종 비교
6. Last block 기준: features[-2:]
7. Seed 1,2,3 반복
8. 기존 BBB EfficientNet-B0 baseline 비교
9. 기존 학습 epoch, early stopping, LR, batch size, augmentation 최대한 유지
```

### 제외한 것

```text
1. CBAM
2. Spatial attention 단독
3. Head hidden dimension 여러 조합
4. Dropout rate 여러 조합
5. 2-hidden-layer 이상의 깊은 MLP head
6. 모든 조합을 한 번에 탐색하는 full grid search
7. AA 공격 분석을 이번 실험 설계에 포함하는 것
```

### 제외 이유

CBAM과 spatial attention은 위치 정보를 함께 활용할 수 있다는 장점이 있지만, MAR20 데이터는 이미 bounding box crop을 통해 항공기 객체 중심으로 구성되어 있다. 따라서 위치를 다시 찾는 spatial attention의 필요성은 상대적으로 낮다고 판단했다.

Head 조합을 과도하게 늘리지 않은 이유는 실험 목적이 classifier head 최적화가 아니기 때문이다. MLP-512 하나만으로도 기존 Linear head 대비 layer depth를 확장했다는 구조 변경 근거가 충분하다.

AA 공격 분석은 중요하지만, 이번 보완 실험에서는 모델 구조와 학습 방식 비교를 먼저 확정하는 것이 우선이다. 공격 분석은 최종 모델이 정해진 뒤 별도 단계로 연결할 수 있다.
